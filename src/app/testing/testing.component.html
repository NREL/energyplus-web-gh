Testing and Validation

Continuous Integration Testing

EnergyPlus uses an extensive set of Continuous Integration tests to ensure that software code fixes to identified issues, as well as new code adding additional features to EnergyPlus, are not causing unexpected changes in the results. This is part of the GitHub workflows that test the software for each new change. The types of tests include:

  • Regression tests run hundreds of EnergyPlus building models and automatically compare the results of the simulations between the new and existing source code.
  • Unit tests that exercise individual or small groups of functions in the source code to ensure that changes to small units of code do not negatively impact the overall results.
  • Build tests that create and compile EnergyPlus and almost all supporting utilities on various platforms (Windows, MacOS, Linux), including documentation and package installers, to ensure that deployments of new versions of EnergyPlus are done smoothly.
  • Miscellaneous tests exercise the multitude of interacting pieces of the entire EnergyPlus package to reduce the unforeseen impacts of changing components on the overall software.

Tracking of the speed and the number of lines and functions that are covered by the tests is also included. Most of these tests can be performed locally by each developer on the team so they can provide the best changes prior to committing them to the common repository.

Current Validation Testing

An important ongoing part of EnergyPlus development is testing using industry-standard methods as major builds are completed. The goal is to make EnergyPlus as bug-free as possible. Currently, the validation primarily follows ASHRAE Standard 140 and is conducted as part of the OpenStudio validation using the same standard. The automated validation system can be seen on the BESTEST-GSR repository. Note that the results are posted under [individual releases]( https://github.com/NREL/BESTEST-GSR/releases) that generally correspond with software releases of OpenStudio and EnergyPlus.

Earlier Validation Testing

Historically, two categories of validation tests were conducted:

  • Analytical tests:
    • HVAC tests, based on ASHRAE Research Project 865
    • Building fabric tests, based on ASHRAE Research Project 1052
  • Comparative tests:
    • ANSI/ASHRAE Standard 140-2011
    • International Energy Agency Solar Heating and Cooling Programme (IEA SHC) BESTest (Building Energy Simulation Test) methods not yet in Standard 140
    • EnergyPlus HVAC Component Comparative tests
    • EnergyPlus Global Heat Balance tests

The BESTest suites compare the results of multiple simulation programs for a series of load-related attributes. For more information on IEA SHC Task 34/Annex 43 (BESTest), visit the IEA Solar Heating and Cooling Programme website.

EnergyPlus testing reports are available for many of these test suites.
